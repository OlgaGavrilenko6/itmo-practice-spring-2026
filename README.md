# ITMO Practice Spring 2026

Исследовательский проект по извлечению **элементарных требований** из нормативных документов с использованием больших языковых моделей (LLM) и агентного подхода.

## О проекте

Цель проекта — автоматизировать обработку нормативно-технической документации (СП, ГОСТ, СанПиН и др.) и преобразовывать текст документа в структурированные данные.

Под **элементарным требованием** понимается атомарная единица нормы:

- одна сущность;
- одно свойство;
- одно ограничение / условие.

# Эксперименты
## Baseline
Файл **baseline.py**

Первый эксперимент представляет собой линейный пайплайн:

DOCX документ разбивается на абзацы -> каждый абзац отправляется в LLM -> модель извлекает требования -> результат сохраняется в JSON.

### Особенности:
- простая архитектура;
- быстрое прототипирование;
- без контекста всего документа;
- без проверки качества ответа.

## Elementary Requirements
Файл **elementary_requirements.ipynb**. Второй эксперимент посвящён улучшению качества декомпозиции требований на атомарные элементы. В эксперимент добавляются извлечение локального и глобального контекстов. 

Под **локальным контекстом** подразумевается информация, без которой атомарное требование может потерять смысл. Под **глобальным контекстом** подраумевается информация, в которой указывается доменная область данного требования, расшифровка сокращенного термина, определение домменого термина. 

### Исследуется:
1) Как лучше формулировать prompt;  
2) Как разбивать сложные предложения; 
3) Как обрабатывать перечисления; 
4) Как выделять локальный и глобальный контекст.

## Agent Experiment
Третий эксперимент реализует agentic workflow на базе LangGraph и LangChain.

Идея: вместо одного запроса используется несколько специализированных агентов:
- агент проверки JSON;
- агент исправления ошибок по комменатриям агента проверки.

### Преимущества:
- выше точность;
- устойчивость к ошибкам модели;
- использование глобального контекста документа;
- возможность масштабирования.

## Архитектура пайплайна
```
DOCX документ
      ↓
Парсинг структуры документа
      ↓
Извлечение абзацев
      ↓
LLM / Agent workflow
      ↓
Элементарные требования
      ↓
JSON dataset
```

## Формат данных
```json
{
  "paragraph": "6.4.8 Пешеходные дорожки на территории ОО следует проектировать...",
  "reqs": [
    {
      "text": "Пешеходные дорожки на территории ОО следует проектировать в соответствии с СП 2.4.3648."
    }
  ]
}
```
