# ITMO Practice Spring 2026

Исследовательский проект по извлечению **элементарных требований** из нормативных документов с использованием больших языковых моделей (LLM) и агентного подхода.

## О проекте

Цель проекта — автоматизировать обработку нормативно-технической документации (СП, ГОСТ, СанПиН и др.) и преобразовывать текст документа в структурированные данные.

Под **элементарным требованием** понимается атомарная единица нормы:

- одна сущность;
- одно свойство;
- одно ограничение / условие.

# Эксперименты

## 1. Baseline Pipeline

Файлы:

Для извлечения элементарных требований без условий применимости: 
```bash
baseline.py
```
Для извечения элементраных требований с условиями применимости:
```bash
baseline_new.py
```

Классический линейный подход:

1. документ разбивается на параграфы – нумерованные списки;
2. каждый параграф отправляется в LLM;
3. LLM извлекает требования и условия применимости по очереди;
4. результат сохраняется в JSON.

Используется как контрольная точка для сравнения остальных экспериментов.

---

## LLM-approach
Файлы:

Для извлечения требований с локальным и глобальным контекстами:
```bash
elementary_requirements.ipynb
```

Для извлечения условий применимости с использованием извлеченного локального и глобального контекстов: 

```bash
elementary_requirements.ipynb
```
Второй эксперимент направлен на улучшение полноты извлечения элементарных требований за счет извлечения локального и глобального контекстов. 

Под **локальным контекстом** подразумевается зависимая от требования информация, без которой элементарное требование может потерять смысл. Под **глобальным контекстом** подраумевается информация, содержащая описание области применимости данного документа: доменная область данного требования, расшифровка сокращенного термина, определение домменого термина. 

### Исследуется:
1) Как лучше формулировать правила извлечения;  
2) Как разбивать сложные предложения; 
3) Как обрабатывать перечисления; 
4) Как выделять локальный и глобальный контекст.

#### Этапы:
1) Документ с извлеченными требованиями отправляется в LLM вместе с разработанными правилами; 
2) LLM отдает результаты – локальный и глобальный контекст каждому требованию;
3) Необходимо отредактирвоать файл и для каждого элементарного требования добавить поля с локальным и глобальным контекстами.

## Agent Experiment
Третий эксперимент реализует LLM-workflow на базе LangGraph и LangChain.

Идея: вместо одного запроса используется несколько специализированных агентов:
- агент проверки локального и глобального контекстов;
- агент исправления ошибок по комменатриям агента проверки.

### Преимущества:
- выше точность;
- устойчивость к ошибкам модели;
- использование глобального контекста документа;
- возможность масштабирования.

## Архитектура пайплайна
```
Извлеченное требование
      ↓
Агент извлечения локального и глобального контекстов
      ↓
Критик, проверяющий извлеченные контексты
      ↓
Редактор, исправляющий локальный и глобальный контексты по комментариям критика
      ↓
JSON-файл
```

## Формат входных данных
```
7.2.2.4	 При проектировании БНК в обособленном здании, размещенном на общем участке с основным зданием ОО, допускается предусматривать сокращенный набор общешкольных помещений. 
По заданию на проектирование возможно исключить административные помещения, сократить число помещений медблока, физкультурно-оздоровительной группы, информационного центра.
```
